# domain-results — Results Compilation & Analysis

## Purpose

Handles automated result compilation for gemma4:e4b test findings.

## Mandatory Rules

1. Every result must reference its validating threshold evidence
2. Progressive analysis — build cumulatively, never delete prior results
3. Synthesis first — aggregate related results before final delivery

## Prohibited

- No standalone results without validation
- No duplicate processing
- No partial outputs

## Documentation Protocol

Create entries in Activity Log.md with timestamp, result summary, JSON output link, and validation cross-reference.
