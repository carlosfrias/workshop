## [S-TIGHT]

B-KR-002 decomposed into 10 sub-steps across 3 waves. **Wave 0** establishes the pre-transparency baseline and commit range. **Wave 1** runs parallel bisection probes across the lab node pool (fnet1–fnet7). **Wave 2** narrows to the exact commit and documents the root cause. Estimated total effort: 4–6 hours, wall-clock: ~60 minutes with full parallelism.

**Bisection strategy:** We know the regression is that `pi-keyword-router` persistently selects `qwen3.5:4b` (low-capacity local) and never escalates to `gemma4:e4b` (medium local) or cloud models (`kimi-k2.6`, `deepseek-v4-pro`). We will use git log of `pi-keyword-router` and `routing-transparency` to identify the commit range, then farm out commit tests to lab nodes.

---

## Bisection Commit Range

**Known bad:** Current HEAD (keyword-router always picks `qwen3.5:4b`)  
**Known good:** Pre-transparency baseline (keyword-router correctly escalated to higher-capacity models)  

**Estimated range:** ~15–40 commits between known-good and known-bad.  
**Lab node advantage:** Each node can test 1 commit independently. With 7 nodes, we can test 7 commits in parallel per round.

**Strategy:** Each lab node checks out 1 commit from the bisection range, runs the regression test, and reports good/bad. The low cloud uses the results to narrow the range and dispatch the next round.
