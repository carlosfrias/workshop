## Architecture Decisions

### Master Prompt System v2.0 — Primary Orchestrator

**Decision:** Master remains the primary orchestrator for all prompts.

**Rationale:**
- More modular design (delegates to packages)
- Health-monitor integration built-in
- Better memory management (on-demand module loading)
- Cleaner separation of concerns

### TI-011 — Lab-Specific Extension

**Decision:** TI-011 becomes a lab-specific extension for multi-node tasks.

**Role:**
- Complexity classification (`classify_prompt.py`)
- Node-aware routing (`ti011_node_registry.py`)
- Multi-node fan-out (`submit_task.py`)
- Performance logging and adaptive feedback

**Trigger Keywords:** `lab`, `cluster`, `fnet`, `multi-node`, `distributed`, `fan-out`

### Routing-Transparency — Unified Audit Layer

**Decision:** Logs all routing decisions from both Master and TI-011.

**Events Tracked:**
- Master: `decomposed`, `delegated`, `health-check`
- TI-011: `classified`, `routed`, `dispatched`
- keyword-router: `routed`, `switched`

**Footer Displays:**
- Route name + matched keywords
- Complexity tier (NEW)
- Delegation target (NEW)
- Node selection (NEW)
- Timing + cost

### Orchestration Patterns

- **Orchestrator fallback** — When agents stall, direct execution is faster

- **Validation early** — Should have tested `--dump` before generating all 7 nodes
