### TI-023 P5 Fix: TUI Thinking Level

**Gap:** TUI showed harness default thinking level, not route-configured thinking level.

**Root Cause:** In `pi-keyword-router/index.ts`, `pi.setThinkingLevel()` was inside the `needsSwitch` block. If the current model already matched the route's model, thinking level was never updated.

**Fix:**
- Removed inner `pi.setThinkingLevel()` call from `needsSwitch` block
- Added unconditional thinking level update after route classification, with `if (result.thinkingLevel !== currentThinkingLevel)` guard
- File: `~/.pi/agent/extensions/pi-keyword-router/index.ts`

**Result:** All routes (including default with `thinkingLevel: "off"`) now correctly set TUI thinking level.
