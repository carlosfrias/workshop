### Key Deviations from Plan

#### 1. Architecture Pivot: Per-Prompt LLM → Hybrid Heuristic

**Planned:** Run qwen3.5:4b for EVERY prompt to classify complexity (3-6s latency)
**Actual:** Built keyword-based heuristic classifier (0-4ms latency, 100% accuracy on test corpus)
**Why:** Per-prompt LLM was catastrophically slow (3-6s) and returned empty responses. The latency made it unusable as a routing layer — every prompt would incur 3-6s before the actual model even started.

**Impact:** This was the right call. A 0ms classifier is viable. A 6s classifier is not.

#### 2. Scope Expansion: Documentation Fixes

**Planned:** Purely technical (classifier + router config)
**Actual:** Added Domain Activation Check, fixed routing ambiguity, expanded AGENTS.md keywords
**Why:** During execution, discovered that "orchestration" keywords didn't trigger technical-infrastructure domain activation. This would have caused the same silent failure in future sessions.

#### 3. Unplanned Addition: Performance Logger

**Planned:** Not mentioned
**Actual:** Built performance_logger.py for tracking routing decisions
**Why:** Realized that without logging, we can't measure Phase 1's impact or improve the classifier over time. This makes Phase 3 (Performance Monitor) partially complete already.
