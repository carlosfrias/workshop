## Cost Impact

With auto-complexity routing active, the following prompt types that previously might have hit cloud via keyword mis-match now stay local:

- "Check if X is online" — was: `infrastructure` (qwen3:8b) → now: `trivial` (qwen3.5:4b) — even cheaper
- "Format this JSON" — was: `infrastructure` via "format" keyword → now: `trivial` (qwen3.5:4b)
- "Write a script for Y" — was: `reasoning` via "analyze" or "create" → now: `simple` (qwen3:8b)
- "Summarize this file" — was: could hit `structured` → now: `trivial` (qwen3.5:4b)

Hard prompts ("Design a framework...") correctly escalate to cloud:
- Before: might have stayed on default (gemma4:e4b) if no strong keyword match
- After: classified as `hard` → `kimi-k2.6` cloud
