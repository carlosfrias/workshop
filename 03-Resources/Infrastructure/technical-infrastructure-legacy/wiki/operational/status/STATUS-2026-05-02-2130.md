## Problem

The Ctrl+P model cycling fix was applied to lab nodes via Ansible but **did not include the orchestrator (Mac)**. The orchestrator still cycled through models not in `models.json` — specifically, the `pi-ollama` extension dynamically registered cloud models (without `:cloud` suffix) under the `ollama` provider, creating duplicates alongside the manually-configured models in `models.json` (with `:cloud` suffix under `ollama` provider).

## Root Cause

1. `pi-ollama` extension in `~/.pi/agent/settings.json` registers models dynamically from API cache
2. Cache models use IDs without `:cloud` suffix (e.g., `kimi-k2.6`) vs models.json using `:cloud` suffix (e.g., `kimi-k2.6:cloud`)
3. Both sets appear in Ctrl+P picker because both providers have auth configured
4. Lab node fix removed extra ollama models; orchestrator needed extension removal + keyword-router config update

## Solution Applied

### 1. Orchestrator Ctrl+P Fix
**Script:** `scripts/fix-orchestrator-ctrl-p.py`

Actions:
- Backed up `~/.pi/agent/settings.json` → `~/.pi/agent/backups/settings.json.fix-orchestrator-ctrl-p.bak`
- Removed `pi-ollama` from `settings.json` packages
- Created `~/.pi/agent/keyword-router.json` with `ollama` provider and `:cloud`-suffixed models (matching `models.json`)

### 2. AGENTS.md Update
- Added "Include orchestrator node (Mac) in infrastructure fixes" to Must Always rules
- Added orchestrator health check (`scripts/orchestrator_health.py`) to Framework Readiness Check

### 3. AGENTS-full.md Update
- Added new section: **Orchestrator Node (Mac) Conventions**
  - Model configuration guidelines
  - Extension management on orchestrator
  - Health monitoring thresholds
  - Ansible inclusion pattern for orchestrator fixes
