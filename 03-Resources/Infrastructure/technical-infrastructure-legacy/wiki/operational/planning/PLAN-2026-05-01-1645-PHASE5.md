# Meta-Orchestration: Continuous Feedback Loop (Phase 5)
**Date:** 2026-05-01
**Status:** Framework implemented, needs 2+ weeks of data

## Overview

The meta-orchestration system improves itself through a continuous feedback loop:

```
1. Execute → 2. Log → 3. Analyze → 4. Update → 5. Repeat
```

## Components

### 1. Execute (Phase 1-2)
- classify_prompt.py routes prompts by complexity
- decompose_task.py breaks down PLANs into sub-tasks
- task-worker.sh executes on nodes
- synthesize_results.py combines outputs

### 2. Log (Phase 3)
- performance_logger.py records every routing decision
- Fields: timestamp, prompt, complexity, model, latency, cost, quality, adequate
- Stored in: wiki/sessions/model-performance-log.json

### 3. Analyze (Phase 4)
- adaptive_feedback.py reads logs weekly
- Detects patterns: mis-routings, reliable models, cost anomalies
- Generates suggestions: downgrade/upgrade model, expand keywords

### 4. Update (Manual — for now)
- Review suggestions from adaptive_feedback.py
- Update classify_prompt.py keyword rules
- Update keyword-router.json routes
- Update AGENTS.md domain keywords
- Commit changes, test, deploy

### 5. Repeat
- Next week: more data → better analysis → tighter routing

## Weekly Ritual (30 minutes every Friday)

```bash
# 1. Generate dashboard
python3 technical-infrastructure/scripts/routing_dashboard.py --weekly --save wiki/status/ROUTING-DASHBOARD-$(date +%Y-%W).md

# 2. Analyze for patterns
python3 technical-infrastructure/scripts/adaptive_feedback.py

# 3. Apply suggested updates (manual review)
# - Edit classify_prompt.py keywords
# - Update .pi/keyword-router.json routes
# - Update AGENTS.md routing table

# 4. Commit and push
git add -A
git commit -m "Weekly routing optimization: $(date +%Y-%m-%d)"
git push origin main
```
