# Planning: Depot Redundancy Model (fnet3 + fnet6)
**Date:** 2026-05-01 09:15  
**Context:** During Ollama deployment. Both fnet3 and fnet6 are Tier 1 with 31GB RAM and identical models.  
**Question:** Should we designate fnet6 as a hot-standby depot, or use both in round-robin for load distribution?  
**Considerations:**
- Network saturation risk if both depots sync simultaneously
- Storage overhead: each depot keeps 3 models (~18GB) — acceptable
- Failover: if fnet3 goes down, nodes need automatic fallback to fnet6
- Sync strategy: push from orchestrator to both, or fnet3→fnet6 replication?
