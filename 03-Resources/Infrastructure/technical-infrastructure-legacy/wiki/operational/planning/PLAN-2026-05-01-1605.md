## Problem

The task distribution system sends commands **TO** nodes and collects command output **FROM** nodes. But a significant amount of work produces **documents and files** that belong in the shared wiki:

- fnet1 LVM rebuild agent produced a detailed narrative → should become `SESSION-NOTES-*.md`
- Ollama deployment on fnet3 produced model inventory → should update `BACKLOG.md`
- Troubleshooting on fnet2 produced decomposition steps → should become `decomposition-examples/`
- Hardware spec extraction produced node JSONs → should feed `lab-capacity-report.json`

Currently these artifacts sit on the node where they were created. They must be:
1. Manually SCP'd back to the orchestrator
2. Manually committed to git
3. Manually linked in WIKI.md

This is friction. Friction means documentation drifts from reality.

## Goal

**Automated artifact collection:** When any node completes a task that produces wiki artifacts, those artifacts flow back to the orchestrator and are committed as part of the normal session workflow — with no additional user effort.

## Design

### Artifact Types
| Type | Example | Destination | How Detected |
|------|---------|-------------|------------|
| Status doc | `STATUS-2026-05-01.md` | `wiki/status/` | Filename matches `STATUS-*` |
| Session notes | `SESSION-NOTES-2026-05-01-0915.md` | `wiki/sessions/` | Filename matches `SESSION-NOTES-*` |
| Recommendation | `RECOMMENDATION-2026-05-01-0915.md` | `wiki/recommendations/` | Filename matches `RECOMMENDATION-*` |
| Plan | `PLAN-2026-05-01-0915.md` | `wiki/planning/` | Filename matches `PLAN-*` |
| Decomposition | `00-decomposition-plan.md` | `wiki/decomposition-examples/*/`* | Path contains `/decomposition-examples/` |
| Backlog update | `BACKLOG.md` | `wiki/operational/` | Filename is `BACKLOG.md` |
| Hardware spec | `fnet1.json` | `lab-specs/` | Filename matches `fnet*.json` |

### Sync Protocol (Extension of task-worker.sh)

After task execution, the worker checks for artifact files in the task's working directory:

```bash
# In task-worker.sh, after writing result to completed/
# Check if task created wiki artifacts
ARTIFACT_TYPES=("STATUS-*" "SESSION-NOTES-*" "RECOMMENDATION-*" "PLAN-*" "BACKLOG.md")
for pattern in "${ARTIFACT_TYPES[@]}"; do
    for file in $(find "$(dirname $task_file)" -name "$pattern" 2>/dev/null | head -10); do
        # Identify destination
        dest=$(classify_artifact "$file")
        if [ -n "$dest" ]; then
            # Sync to orchestrator
            scp "$file" "orchestrator:$dest/" 2>/dev/null || true
        fi
    done
done
```

### Artifact Classifier

```bash
classify_artifact() {
    local file="$1"
    local basename=$(basename "$file")
    
    case "$basename" in
        STATUS-*) echo "/home/friasc/Dropbox/workshop/technical-infrastructure/wiki/status" ;;
        SESSION-NOTES-*) echo "/home/friasc/Dropbox/workshop/technical-infrastructure/wiki/sessions" ;;
        RECOMMENDATION-*) echo "/home/friasc/Dropbox/workshop/technical-infrastructure/wiki/recommendations" ;;
        PLAN-*) echo "/home/friasc/Dropbox/workshop/technical-infrastructure/wiki/planning" ;;
        BACKLOG.md) echo "/home/friasc/Dropbox/workshop/technical-infrastructure/wiki/operational" ;;
        fnet*.json) echo "/home/friasc/Dropbox/workshop/technical-infrastructure/lab-specs" ;;
        *)
            # Check if parent directory indicates decomposition
            if [[ "$file" == *"/decomposition-examples/"* ]]; then
                echo "/home/friasc/Dropbox/workshop/technical-infrastructure/wiki/decomposition-examples"
            fi
            ;;
    esac
}
```

### Orchestrator Commit Automation

When artifacts arrive on the orchestrator, a small script commits them:

```bash
#!/bin/bash
# wiki-auto-commit.sh — called by inotify or polling script on orchestrator

cd /home/friasc/Dropbox/workshop

# Check for new/modified files
new_files=$(git status --short | grep -E "wiki/|lab-specs/" | awk '{print $2}')

if [ -n "$new_files" ]; then
    echo "$new_files" | while read f; do
        git add "$f"
    done
    
    git commit -m "Auto-collect wiki artifacts from lab nodes

$(git diff --cached --stat | grep -E "wiki/|lab-specs/")
" && git push origin main
fi
```

### Phase 1: Manual-Triggered Sync (Immediate, Low Risk)

The worker script just **makes artifacts available**. An orchestrator command pulls them:

```bash
# User runs this after a session
cd technical-infrastructure/scripts
./task-collect-artifacts.sh --nodes all
./wiki-auto-commit.sh
```

This is what we should do **now**.

### Phase 2: Auto-Sync (Future, Higher Risk)

The worker SCPs artifacts immediately after task completion. A file watcher on the orchestrator auto-commits.

Requires:
- Reliable SSH key auth from every node → orchestrator (reverse direction)
- File watcher (launchd/fswatch) on orchestrator consuming CPU
- Conflict resolution when multiple nodes produce simultaneous BACKLOG.md updates

### Phase 3: Git-Driven Sync (Ideal)

Each node runs a git client. Work is done on branches. Node pushes branch to orchestrator. Orchestrator merges. This eliminates SCP entirely.

Requires:
- pi sessions or agent processes that can write to a git branch
- Merge conflict handling for overlapping artifacts
- Node identity (who authored this commit?)

## Relationship to Orchestration Plan

This is **Phase 2** of the task distribution system. Phase 1 (task-worker.sh) sends commands **TO** nodes. Phase 2 (artifact sync) collects documentation **FROM** nodes.

Both use the same infrastructure (SSH/NFS) but serve different purposes. Phase 1 is operational. Phase 2 is documentation.
