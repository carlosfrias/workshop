# CORE — Import Domain Identity & Reference

**Section ID:** `import-core`
**Size:** ~2.5KB | **LOD:** Low | **Load:** Always (first section)

Six-stage pipeline: Inbox → Validate → Pending → Review → Import → Archive. Every import creates a timestamped backup in `ledger/.backups/`. Patch files enable per-record corrections via Trade ID. Rollback restores pre-import state.

---

## Conventions

- **Pipeline stages**: `staging/{inbox,pending,approved,rejected}/`
- **Trade ID format**: `#SCHWAB-YYYYMMDD-NNN` (generated by pipeline)
- **Backups**: `ledger/.backups/main_{timestamp}.beancount`
- **Logs**: `logs/pipeline.log` + `logs/import_jobs.jsonl`
- **All monetary values**: USD

## Supported Formats

| Format | Status | Library |
|--------|--------|---------|
| PDF (Schwab statements) | Implemented | `pdfplumber` |
| Microsoft Excel (XLSX) | Implemented | `openpyxl` |
| CSV | Implemented | Built-in |
| LibreOffice Calc (ODS) | Pending native | Routed through XLSX parser |
| Quicken (QIF/QFX) | Not yet | — |
| Futures contracts (Schwab) | Not yet | Different layout than equity/options |
| Daily Schwab email notifications | Not yet | — |
