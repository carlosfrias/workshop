# Session Journal — 2026-05-29-2325

## Summary

**Final fix for fleet idle CPU burn.** Identified root cause: fleet nodes missing `pi-model-router` extension and `defaultModel` setting. Pi's fallback chain auto-selected first model from models.json, loading it into Ollama GPU at startup. Also deployed fleet-cooling systemd oneshot for fan/governor persistence. Full standup validated across all 7 nodes post-reboot.

## What Changed

### Root Cause Discovery
- pi auto-loads a model via `buildSessionOptions` when `enabledModels` exist and `defaultModel` is absent → selects `enabledModels[0]`
- Second fallback: `findInitialModel` in sdk.js calls `modelRegistry.getAvailable()[0]` when no defaultModel is set
- Fleet nodes had no `pi-model-router` extension (`npm:@yeliu84/pi-model-router`) to manage model selection
- Prior fix (removing `--model`, `defaultModel`, initial prompt) was incomplete — only fixed the first layer

### Fix Applied
- Installed `npm:@yeliu84/pi-model-router` on all 7 fleet nodes
- Set `defaultModel: "openbmb/minicpm-o2.6:8b"` (medium tier from model-router)
- Created `fleet-cooling.service` (systemd oneshot) for fan + CPU governor persistence
- Updated Ansible: renamed `deploy-model-router.yml` → `phase2.5-model-router.yml`, added to Chain 2 in standup-fleet-chains.yml
- Added fleet-cooling deployment to `phase5-agent-services.yml`
