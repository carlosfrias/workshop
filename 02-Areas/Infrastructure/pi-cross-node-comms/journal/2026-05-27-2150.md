## Key Decisions

1. **`sender_node` defaults to `"?"`** when missing from prompt event — matches existing `senderCwd` pattern. Display format omits `[?]` prefix for unknown nodes, preserving backward compatibility.
2. **No changes to `coms_net_send` / `coms_net_await`** — those are outbound APIs where the sender identity is already known locally. The `sender_node` gap only affected **inbound** prompt display.
3. **Server `agent_updated` now includes `node`** — previously only sent changed fields (`context_used_pct`, `queue_depth`, `model`, `status`). Node is now always included in case it was updated by re-registration.
